3、遇到的问题及解决方案

	首页banner图片的处理
	
	    描述：在不同尺寸下图片展示的位置不同

	    解决：

		1. 在不同尺寸下使用不同的图片；
	
		    优点： 图片撑开父容器高度，高度自适应；

		    缺点： 浏览器在1个尺寸下加载所有尺寸下的图片，冗余；

		2. 使用背景颜色与背景图结合，在不同尺寸下设定不同的位置以及背景图尺寸；

		    优点： 只需要加载一张图片；
	
		    缺点： 高度需要逐个调整，media代码增多，需要适配多个尺寸下背景图尺寸；


	
	在使用百分比布局过程中margin值会影响对百分比的计算

	    解决：使用怪异盒模型，并使用padding代替margin值来设置元素之间的间隙；


4、项目描述

	响应式项目，根据主流分辨率，对不同分辨率屏幕做了适配（320,768,1024,1200,1400+）；

	分辨率适配处理：

		1) 利用@media做适配，在不同分辨率下加载相应的样式；

		2) 设置了页面最小宽度，避免在页面尺寸极端小的情况下，影响页面的展示效果；

	布局内容结构化，使用与内容相关的命名，便于二次开发与修改；

	使用 html5 新增标签，遵从标签语义化，关爱浏览器，优化SEO；

	使用了CSS3样式，优化页面交互体验；如：

		1) 使用 animation 与 transform 等实现 banner 图轮播；

		2) 使用 transform:scale();对小图标进行相应缩放，优化展示效果；

		3) 使用 transform:rotate();对右侧反馈按钮进行调整；

		4) 使用 transition 优化 hover的动画效果；

	使用了新属性 placeholder 处理 input 中的预留文字；

	使用CSS样式写出的简单的小图标，减少对图片的使用；

	多处使用了百分比 % 布局，结合多怪异盒模型的应用；


5、该项目的技术总结

	在响应式项目中，在初期对页面的布局与PC端是一样的，有时需要更多参考一下在不同分辨率下的表现；使用媒询虽然可以在不同尺寸下改变布局，但是仍然要注意在布局过程中考虑不同分辨率的状态，优化代码结构；
	
	理解对 @media 不同分辨率下的适配应用；

	在布局过程中使用百分比布局，结合着怪异盒模型使用会更加便捷；

	做适配过程中需要同时注意到页面的每个地方的细节，因为由于尺寸的变化会影响到整体的布局位置；
